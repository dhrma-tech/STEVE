# Database Schema

## Schema Status
Frozen after Phase 1. Any change after this document is completed requires:
- A migration file.
- A `DECISIONS.md` entry.
- `SCRATCHPAD.md` drift note.
- Checkpoint mention.

## Conventions
- Primary keys: string ids generated by application (`cuid`/uuid compatible).
- Timestamps: `createdAt`, `updatedAt`; soft deletions use `deletedAt`.
- JSON fields are used for provider payloads, flexible settings, and generated content that does not need relational querying.
- All organization-scoped tables include `organizationId`.
- Provider secrets store metadata only unless explicitly encrypted; write-only secrets are never returned by API.

```text
users 1--many memberships many--1 organizations
organizations 1--many departments
organizations 1--many agents
departments 1--many agents
departments 1--many tasks
agents 1--many tasks
tasks 1--many subtasks
tasks 1--many task_sessions

organizations 1--many chat_threads
chat_threads 1--many chat_messages
organizations 1--many folders
folders 1--many files
files 1--many file_versions
organizations 1--many roadmap_stages

roadmap_items many--many roadmap_items through roadmap_dependencies
roadmap_items 1--many tasks
organizations 1--many integrations

organizations 1--many secrets

agents 1--many agent_inboxes

organizations 1--many audit_logs
tasks 1--many approvals
agent_actions 1--many approvals
```
